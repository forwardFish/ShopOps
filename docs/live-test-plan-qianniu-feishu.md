# ShopOps 千牛 PC 到飞书多维表格测试计划

## 目标

验证 ShopOps 能每 10 分钟刷新千牛 PC 页面，采集：

- `推广中心` 页面经营概览里的 `花费`
- `已卖出宝贝` 页面可见订单数据

并通过代码写入飞书多维表格。验收必须用同一轮采集的千牛页面截图与飞书多维表格截图逐项对比，通过后才算完成。

## 飞书表结构

当前只创建两个业务表，使用飞书 OpenAPI 代码创建，不使用 MCP。

| 表名 | 用途 | 必要字段 |
| --- | --- | --- |
| `订单明细原始表-千牛淘宝` | 保存已卖出宝贝页面订单行 | `采集时间`, `订单号`, `创建时间`, `买家昵称`, `商品名称`, `单价`, `数量`, `履约/售后状态`, `交易状态`, `实收款`, `操作信息`, `原始数据`, `页面截图` |
| `推广数据表-千牛淘宝` | 保存推广中心花费 | `采集时间`, `花费`, `页面URL`, `页面截图` |

## 前置条件

1. `.env` 中存在 `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 或 `APP_ID` / `APP_SECRET`。
2. 飞书自建应用已开通多维表格读写/管理权限。
3. 如果使用现有多维表格，`.env` 中填入 `FEISHU_APP_TOKEN`。
4. 如果没有现有多维表格，运行 bootstrap 脚本创建新 Base。
5. 千牛 PC 已登录，且本机 CDP 地址 `http://127.0.0.1:9222/json/version` 可访问。
6. `.env` 中设置：

```text
ORDER_SOURCE=crawler
PROMOTION_SOURCE=qianniu_pc
USE_MOCK_COLLECTORS=false
STORAGE_BACKEND=feishu
FETCH_INTERVAL_SECONDS=600
QIANNIU_CDP_URL=http://127.0.0.1:9222
FEISHU_PLATFORM_NAME=千牛淘宝
EVIDENCE_DIR=docs/live-evidence
```

## 建表步骤

不使用 MCP，直接通过飞书 OpenAPI 建表：

```powershell
python scripts\bootstrap_feishu_tables.py --platform 千牛淘宝 --write-env .env
```

预期结果：

- 如果 `.env` 中没有 `FEISHU_APP_TOKEN`，脚本创建新的多维表格 Base，并把 `FEISHU_APP_TOKEN` 写回 `.env`。
- 脚本创建 `订单明细原始表-千牛淘宝` 和 `推广数据表-千牛淘宝`。
- 脚本把 `FEISHU_TABLE_ORDERS_RAW` 和 `FEISHU_TABLE_PROMOTION_SNAPSHOT` 写回 `.env`。

## 单轮采集测试

执行一次采集：

```powershell
python -c "from shopops.scheduler import Scheduler; Scheduler().run_once()"
```

验收点：

- 千牛推广中心截图中的 `花费` 与飞书 `推广数据表-千牛淘宝` 的 `花费` 一致。
- 千牛已卖出宝贝截图中的订单行与飞书 `订单明细原始表-千牛淘宝` 一致。
- 失败时不写 0 伪数据，应写失败状态或保留空值。

## 10 分钟循环测试

启动验收脚本：

```powershell
python scripts\run_live_acceptance.py --cycles 3
```

至少观察 3 轮：

| 轮次 | 时间 | 要验证的内容 |
| --- | --- | --- |
| 第 1 轮 | T+0 | 推广花费和订单数据首次写入飞书 |
| 第 2 轮 | T+10 分钟 | 页面已刷新，新增一轮采集时间记录 |
| 第 3 轮 | T+20 分钟 | 再次刷新页面，飞书有第三轮记录 |

每轮都必须保存：

- 千牛推广中心截图
- 千牛已卖出宝贝截图
- 飞书推广数据表截图
- 飞书订单明细原始表截图
- 程序侧验收 JSON：`docs/live-evidence/acceptance-run-*.json`
- 对比结果记录

## 对比规则

推广中心：

| 千牛字段 | 飞书字段 | 通过条件 |
| --- | --- | --- |
| `花费` | `花费` | 数值一致 |
| 当前采集时间 | `采集时间` | 在本轮采集时间窗口内 |

已卖出宝贝：

| 千牛字段 | 飞书字段 | 通过条件 |
| --- | --- | --- |
| 订单号 | 订单号 | 完全一致 |
| 创建时间 | 创建时间 | 完全一致 |
| 买家昵称 | 买家昵称 | 页面展示值一致 |
| 商品名称 | 商品名称 | 页面展示值一致或可解释截断 |
| 单价 | 单价 | 数值一致 |
| 数量 | 数量 | 数值一致 |
| 履约/售后状态 | 履约/售后状态 | 文案一致 |
| 交易状态 | 交易状态 | 文案一致 |
| 实收款 | 实收款 | 数值一致 |
| 操作区可见信息 | 操作信息 | 页面展示信息一致或覆盖主要动作 |

## 完成标准

只有同时满足以下条件，最终状态才能判定为 `PASS`：

1. 飞书 Base/表由代码创建或由代码确认并写入。
2. 每 10 分钟刷新采集至少连续跑 3 轮。
3. 飞书表中存在同一轮采集的数据。
4. 千牛 PC 页面截图与飞书表截图逐项对比通过。
5. 推广失败、订单失败、飞书写入失败时没有写入 0 伪数据。

缺少千牛 PC 会话、飞书授权或真实截图对比证据时，不能判定为完成。
